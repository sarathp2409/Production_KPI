     Historical Pagination Data CSV Capture Plan

     Objective

     Design a CSV-based data collection system for capturing 2 years of 
     historical pagination data from 14 plants, with per-plant CSV files for
      ease of distribution and data entry by different location 
     coordinators.

     Data Structure Analysis

     Models to Populate

     1. PaginationReport - One record per plant per issue_date
     2. GNPInformation - One-to-one with PaginationReport
     3. BookDetails - Multiple books per report (TOI B-1, TOI B-2, TIMS B-1,
      TIMS B-2, TOI Supp-1/2/3)
     4. CompetitorData - Multiple competitors per report
     5. CompetitorBookDetails - 2 book types (Main, Supplement) per 
     competitor

     Plant Configuration

     14 plants with varying GNP publication configurations:
     - Plants WITH ET publications: Bhosari, Bommasandra, Chemmencherry, 
     Chinhat, Kandivali, Manesar, Nacharam, Sahibabad, Saltlake, Vejalpur
     - Plants WITHOUT ET: Airoli, Baroda, Butibori, Trivandrum

     ---
     Recommended CSV Structure

     Approach: 2 CSV Files Per Plant

     Each plant will receive 2 CSV template files:
     1. {plant_name}_inhouse.csv - For books/pagination data
     2. {plant_name}_competitor.csv - For competitor data

     Rationale:
     - Separates in-house data from competitor data (different complexity 
     levels)
     - Matches existing import script architecture
     - Easier for users to understand and fill
     - Can import in-house data first, then competitors (dependency order)

     ---
     CSV Template Specifications

     1. In-House Data CSV ({plant_name}_inhouse.csv)

     Structure: One row per book per date (denormalized for ease of entry)

     Supp-2,TOI Supp-3"',
         allow_blank=False,
         showDropDown=False,  # False = show dropdown arrow

         errorTitle="Invalid Value",
